# Don't Touch Button — 代码规范

## 1. 项目结构

```
res://
├── scenes/          # 场景文件 (.tscn)
│   ├── actors/      # 玩家、敌人、NPC 等实体场景
│   ├── ui/          # UI 场景
│   ├── levels/      # 关卡/地图场景
│   └── props/       # 道具、可交互物
├── scripts/         # 非场景绑定的 GDScript (.gd)
│   ├── autoload/    # 全局单例
│   ├── components/  # 可复用组件（通过 @export 注入到场景节点）
│   └── utilities/   # 纯工具函数/类
├── assets/          # 美术资源
│   ├── sprites/
│   ├── sounds/
│   ├── fonts/
│   └── shaders/
└── docs/            # 文档（不会被 Godot 导入）
```

规则：
- `scenes/` 下每个 `.tscn` 自带根节点脚本（不拆到 scripts 目录），除非脚本被多个场景共享
- 共享脚本放在 `scripts/components/` 或 `scripts/autoload/`

## 2. 命名规范

### 2.1 文件命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 场景 | `snake_case.tscn` | `player_character.tscn` |
| 脚本 | `snake_case.gd` | `player_controller.gd` |
| 资源 | `snake_case.tres` | `red_material.tres` |
| 纹理 | `snake_case.png` | `player_idle.png` |
| 着色器 | `snake_case.gdshader` | `outline_effect.gdshader` |

### 2.2 代码命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 类名 | PascalCase | `class_name PlayerController` |
| 函数/方法 | snake_case | `func take_damage(amount: int):` |
| 变量 | snake_case | `var max_health: int` |
| 常量 | UPPER_SNAKE | `const MAX_SPEED = 400.0` |
| 信号 | snake_case | `signal health_changed(new_health)` |
| 枚举值 | UPPER_SNAKE | `enum State { IDLE, RUNNING, JUMPING }` |
| 私有成员 | `_snake_case` 前缀下划线 | `var _current_health: int` |
| 私有方法 | `_snake_case` 前缀下划线 | `func _calculate_damage():` |

### 2.3 节点命名

- 场景根节点使用场景名的大驼峰形式：`PlayerCharacter`
- 子节点使用 PascalCase 描述性名称：`HealthBar`、`CollisionShape3D`、`AnimationPlayer`
- 禁止保留默认名称如 `Node2D`、`Sprite3`、`ColorRect2`
- 同类节点用数字后缀：`EnemySpawnPoint1`、`EnemySpawnPoint2`

## 3. GDScript 编写规范

### 3.1 代码结构顺序（单文件内）

```gdscript
class_name MyClass
extends Node

# 1. 信号
signal something_happened

# 2. 枚举
enum State { IDLE, ACTIVE }

# 3. 常量
const GRAVITY = 9.8

# 4. @export 变量
@export var speed: float = 200.0
@export var max_health: int = 100

# 5. public 变量
var is_active: bool = true

# 6. private 变量
var _current_health: int

# 7. @onready 变量
@onready var sprite: Sprite3D = $Sprite3D
@onready var animation_player: AnimationPlayer = $AnimationPlayer

# 8. 生命周期方法
func _ready():
    pass

func _process(delta: float):
    pass

func _physics_process(delta: float):
    pass

# 9. 信号回调
func _on_area_entered(area: Area3D):
    pass

# 10. public 方法
func take_damage(amount: int):
    pass

# 11. private 方法
func _apply_knockback(direction: Vector3):
    pass
```

### 3.2 类型提示

**所有变量、参数、返回值必须有类型提示：**

```gdscript
# 好
var speed: float = 200.0
func deal_damage(target: Node3D, amount: int) -> bool:
    return true

# 坏
var speed = 200.0
func deal_damage(target, amount):
    return true
```

例外：仅在 `_ready()` 中的临时变量可省略。

### 3.3 空行与缩进

- 缩进使用 Tab（遵循 `.editorconfig`）
- 逻辑块之间空一行（函数之间、类定义之间、大的逻辑段落之间）
- 相关变量声明不空行，不相关变量组之间空一行

### 3.4 字符串

- 用户可见字符串使用 `tr()` 包裹，为后续本地化做准备：
```gdscript
label.text = tr("Press Start")
```
- 格式化使用字符串插值：`"Health: %d/%d" % [current, max_health]`

### 3.5 条件判断

- 避免深层嵌套：早返回优于深嵌套
```gdscript
# 好 — 早返回
func process_target(target: Node) -> void:
    if not is_instance_valid(target):
        return
    if not target.is_alive:
        return
    _apply_damage(target)

# 坏 — 深层嵌套
func process_target(target: Node) -> void:
    if is_instance_valid(target):
        if target.is_alive:
            _apply_damage(target)
```

### 3.6 注释

- **不要注释"做了什么"** — 代码应该自解释
- **只注释"为什么这样做"** — 非显而易见的业务逻辑、设计决策、workaround
```gdscript
# 好
# Jolt Physics 在高帧率下对此碰撞体类型有已知问题，降至 30fps 检测频率
func _physics_process(delta: float) -> void:
    pass

# 坏
# 设置速度
speed = 200.0
```

### 3.7 信号连接

- 信号回调统一命名为 `_on_<source_node>_<signal_name>`
```gdscript
func _on_button_pressed() -> void:
    pass
func _on_player_health_changed(new_health: int) -> void:
    pass
```
- 代码连接的信号在 `_ready()` 中完成，统一放在一起

## 4. 场景设计规范

### 4.1 单一职责

每个场景只负责一件事。如果描述场景需要"和"字，考虑拆分：
- 好：`PlayerController.tscn`（只控制输入 → 移动）
- 坏：`PlayerAndUI.tscn`

### 4.2 场景耦合

- **向下调用，向上发信号**
- 父节点可以直接调用子节点方法
- 子节点通过信号通知父节点
- 同级节点通过共同祖先节点中转，或者用 Autoload 信号总线

### 4.3 @export 注入依赖

用 `@export` 注入外部依赖而非硬编码路径：
```gdscript
# 好 — 注入依赖
@export var hurtbox: Area3D
@export var health_bar: ProgressBar

# 坏 — 硬编码路径
var hurtbox: Area3D = $"../../../Hurtbox"
```

## 5. Autoload 规范

Autoload 只用于：
- 跨场景的全局状态（`GameState`）
- 事件总线（`EventBus`）
- 工具函数库（`Utils`）

Autoload 不应该持有场景节点的直接引用。场景节点通过信号与 Autoload 通信。

## 6. 性能偏好

- 优先使用 `CharacterBody3D` + 物理而非手动碰撞检测
- 大量同类对象用 `MultiMeshInstance3D` 而非逐个实例化
- Tween 用 `create_tween()` 而非 `Tween` 节点（减少场景树节点数）
- 避免在 `_process()` 中 `get_node()`，用 `@onready` 缓存

## 7. 版本控制

- `.godot/` 已在 `.gitignore`
- 资源文件使用 `.tres`（文本格式）优于 `.res`（二进制），便于 diff
- 场景依赖的资源放入 `assets/` 后再引用，不直接引用外部路径

## 8. 开发流程偏好

- **先跑通，后优化**：优先验证玩法可行性，不过早优化性能
- **场景测试**：每个独立场景应该能单独运行（按 F6），不依赖启动关卡
- **迭代粒度**：单次改动尽量小，能让场景跑起来就提交一次
- **信号优先**：新增交互优先考虑"发信号 → 外部响应"模式，保持场景独立可测
